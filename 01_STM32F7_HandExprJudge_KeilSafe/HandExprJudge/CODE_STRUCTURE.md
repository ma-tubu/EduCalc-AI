# HandExprJudge 代码结构说明

本工程已经将原来集中在 `main.c` 的业务逻辑拆分为多个模块，便于调试、答辩展示和实验报告撰写。

## 主控入口

- `Core/Src/main.c`
  - 完成 HAL 初始化、系统时钟配置、GPIO/CRC/USART 初始化。
  - 重定向 `printf` 到 USART1。
  - 启动串口空闲中断接收。
  - 在主循环中调用 `App_ProcessPendingFrame()` 处理收到的一帧数据。

## 应用状态机与串口协议

- `Core/Inc/app_controller.h`
- `Core/Src/app_controller.c`
  - 管理串口接收缓冲区。
  - 解析串口输入的数字 payload。
  - 支持三种输入格式：
    - `784*N`：固定 28x28 字符块输入。
    - `28*W`：固定高度 28 的整行图像输入，板端自动分割。
    - `-1 width height pixels...`：任意宽高图像输入，板端按宽高解析并分割。
  - 管理识别流程状态：
    - 等待新表达式。
    - 低置信度字符重写。
    - 多个低置信度字符选择重写下标。
    - 答案错误后只重写答案部分。
  - 输出识别字符、置信度、表达式字符串和判题结果。

## Cube.AI 推理封装

- `Core/Inc/ai_infer.h`
- `Core/Src/ai_infer.c`
  - 初始化 Cube.AI 网络。
  - 调用 `ai_network_run()` 完成单字符推理。
  - 对 softmax 输出做 `argmax`。
  - 返回类别编号和最大概率作为置信度。
  - 维护 15 类标签表：`0-9`、`+`、`-`、`*`、`/`、`=`。

## 图像预处理与字符分割

- `Core/Inc/image_segment.h`
- `Core/Src/image_segment.c`
  - 将输入像素归一化到 `0.0~1.0`。
  - 根据输入格式选择处理路径。
  - 对整行图像执行按列投影分割。
  - 自动检测每个字符的外接矩形。
  - 将字符裁剪、缩放、居中填充为 `28x28`，再送入模型。

## 表达式解析与智能判题

- `Core/Inc/expr_judge.h`
- `Core/Src/expr_judge.c`
  - 解析形如 `A+B=C`、`A-B=C`、`A*B=C`、`A/B=C` 的表达式。
  - 提取左操作数、运算符、右操作数和用户答案。
  - 在 STM32F7 板端计算正确答案。
  - 判断用户答案是否正确。
  - 返回答案起始位置，便于答案错误时只重写结果部分。

## 全局配置

- `Core/Inc/app_config.h`
  - 集中管理串口缓冲区大小、最大字符数、低置信度阈值等参数。
  - 目前低置信度重写阈值为 `0.70`，警告阈值为 `0.85`。

## 报告中可以强调的实现点

- 不是单字符识别，而是完整表达式识别与判题。
- 板端完成字符分割、补齐、识别、置信度判断和表达式计算。
- 支持错误答案重写和低置信度交互重写。
- 工程按功能模块管理，便于维护和展示嵌入式软件设计思路。

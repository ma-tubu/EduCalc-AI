# 2026-06-25 编号测试数据说明

这些文件用于 XCOM 串口调试助手中复制发送。发送前请确认串口波特率和板端程序一致，且一次只打开一个串口工具。

## 01 单字符识别

- `01_single_char_8_784.txt`
- 输入格式：`784` 个数字，单个 `28x28` 字符。
- 预期用途：验证最基础的单字符推理链路。

## 02 固定长度 784*N 表达式

- `02_fixed784_3plus2eq5.txt`
- 表达式：`3+2=5`
- 输入格式：`784*5 = 3920` 个数字。
- 预期用途：验证固定 28x28 字符块的多字符识别和正确判题。

## 03 固定 28 高度，板端自动分割

- 预期用途：验证板端按列投影自动分割。
- 预览文件：`03_preview_line28_3plus2eq5.pgm`

## 08 自然低置信度案例

- `08_lowconf_natural_9mul8eq64_fixed784.txt`
- 表达式：`9*8=64`
- 输入格式：`784*6 = 4704` 个数字。
- 预期用途：利用数字 `9` 常见低置信度，触发“书写不清晰，请重写”提示。

## 09 人工低置信度案例

- `09_lowconf_artificial_3plus_blank_eq5_fixed784.txt`
- 表达式设计：`3 + 空白 = 5`
- 输入格式：`784*5 = 3920` 个数字。
- 预期用途：人工构造第 2 号字符低置信度或识别异常，测试交互重写。

交互步骤：

1. 发送 `09_lowconf_artificial_3plus_blank_eq5_fixed784.txt`。
2. 如果系统提示低置信度，发送命令 `0`。
3. 如果系统要求选择下标，发送 `2`。
4. 发送 `10_rewrite_char2_as_2_784.txt`。
5. 预期系统重新识别为 `3+2=5` 并判定正确。

## 10 低置信度字符重写 payload

3. 发送 `12_rewrite_answer_55_1568.txt`。
4. 预期系统重新识别为 `11*5=55` 并判定正确。
